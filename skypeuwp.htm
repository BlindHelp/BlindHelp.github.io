Utilisation des scripts JAWS pour Skype UWP
Dernière révision en novembre 2017
La traduction à été fait par Rémy Ruiz (BlindHelp) en novembre 2017.

Ce guide fournit des conseils sur l'utilisation de Skype UWP, également appelé Skype pour Windows 10, avec JAWS pour Windows et ces scripts. Lorsque les scripts sont installés et que Skype UWP est ouvert et mis en focus, ce document peut être ouvert via un double appui sur TOUCHE JAWS+F1 (ou Insert+F1).

Notez que Skype UWP fournit beaucoup de raccourcis clavier qui ne nécessitent pas de scripts. Ces scripts simplifient certaines tâches et fournissent quelques raccourcis non fournis par Skype UWP lui-même.

Configuration système requise pour les utilisateurs de JAWS

Pour un fonctionnement correct avec Skype UWP, ces scripts nécessitent JAWS 18.0 ou supérieur. (Les versions de JAWS antérieures à 18.0 ne prennent pas en charge les applications du script UWP) (Universal Windows Platform).

Instructions d'installation du script

Pour installer ces scripts sur un nouveau système:

  1. Charger JAWS si cela n'a pas déjà été fait. Cela nécessitera des privilèges administrateur sur l'ordinateur.
  2. Exécutez JAWS en tant qu'utilisateur pour lequel les scripts doivent être installés. Ceci et les étapes suivantes doivent être effectuées pour chaque utilisateur de l'ordinateur qui utilisera JAWS avec ces scripts.
  3. Exécutez le programme d'installation pour ces scripts et suivez les instructions à l'écran. Veillez à installer les scripts dans la Version JAWS si une liste de versions JAWS est présentée.
  4. Pour vérifier la réussite de l'installation, tapez Insert+Q pendant que Skype UWP est mis en focus. Une partie de la réponse verbale de JAWS devrait être un numéro de révision. Si vous n'entendez pas de numéro de révision, les scripts ne sont pas correctement chargés.

Configuration de Skype UWP à utiliser avec JAWS

Les étapes suivantes peuvent améliorer les fonctionnalités de Skype avec JAWS et ces scripts. Ces éléments sont facultatifs et soumis à des préférences personnelles.

  • Activer le son de discussion (chat en anglais) pour les discussions entrantes dans la fenêtre en cours: Cette fonctionnalité est désactivée par défaut mais peut être activée si vous préférez:
    1. Frapper Ctrl+, (virgule) pour ouvrir le panneau Paramètres.
    2. Tab dans la case à cocher "Lire le son pour la messagerie instantanée entrante dans la discussion en cours" et, si elle n'est pas cochée, appuyez sur Espace pour la cocher.
    3. Appuyez sur ECHAP pour fermer le panneau Paramètres. (Dans Skype UWP, Les modifications de Paramètres sont immédiates; donc en ferment le panneau avec ECHAP ne retournent pas aux modifications apportées.)
  • Désactiver les aperçus de liens Web (procédure similaire à celle ci-dessus).
Avertissement: Il est nécessaire de répéter ces étapes de configuration après la mise à jour vers une autre version de Skype UWP. (Cette situation s'est produite dans onze jours au cours du développement du script en octobre 2017, mais ne devrait normalement pas se produire.)

Utiliser Skype UWP avec JAWS et les scripts

Cette section et ses sous-sections fournissent des astuces pour utiliser Skype UWP avec JAWS et ces scripts.

Description de l'écran Skype UWP

Skype UWP est structuré différemment de Skype pour Windows Desktop:

  • Il n'y a pas de barre de menus et il n'y a pas de barre d'outils ni de ligne d'état. (Il existe toutefois de nombreux raccourcis clavier, voir la sous-section suivante.)
  • Tout est contenu dans une fenêtre unique. Il peut y avoir des fenêtres émergentes ou des sections déroulantes pour cette fenêtre, mais Alt+Tab ne trouvera qu'une seule tâche pour Skype UWP. Alt+Espace fait toujours apparaître un menu Système à partir duquel il est possible de réduire, agrandir, restaurer ou fermer la fenêtre Skype UWP.

La liste de conversation est un élément central de la fenêtre Skype UWP et est toujours affichée sauf si elle est couverte par une fenêtre émergente. Il y a une liste de contacts, mais il apparaît sur la fenêtre principale lorsque cela est demandé.

Il y aura une conversation à l'écran à un moment donné. Si aucune conversation n'a été affichée depuis le lancement de Skype, une section Home (Accueil en français), est affichée à la place. Cette section n'est pas une section HTML comme c'était le cas dans le client classique Desktop, mais Tab atteindra les contrôles à l'intérieur.

Raccourcis clavier natifs de Skype UWP

Skype UWP fournit plusieurs raccourcis clavier. Celles-ci fonctionnent avec ou sans JAWS ou ces scripts. Une grande partie de cette information apparaît également sur Microsoft (page en anglais) "What are hotkeys and how do I use them in Skype?" cette page en anglais , qui répertorie également les raccourcis clavier pour d’autres versions de Skype sur Windows et d’autres systèmes d’exploitation. Voir aussi sur Microsoft (page en anglais) Keyboard shortcuts in Windows page pour les raccourcis fournis par Windows mais pertinents pour Skype UWP. Le plus notable parmi ceux-ci est Windows+V, qui met le focus sur les notifications et permet ainsi d'accéder aux boutons ID du correspondant (Autrement dit l'appelant), et Répondre pour un appel Skype entrant.

Commandes et fonctionnalités du script JAWS

Les commandes du script Skype UWP sont des commandes par frappes successives (commandes séquentielles), plutôt que des combinaisons multi-touches. Cela permet d'éviter les collisions entre les commandes par frappes successives (commandes séquentielles), du script et les séquences de touches d'application natives. Toutes ces séquences de touches commencent par une touche de préfixe commune, crochet ouvert ([) par défaut. Les utilisateurs peuvent modifier la touche de préfixe si nécessaire en tapant crochet ouvert [ suivi par TOUCHE JAWS+C ou Insert+C. Le changement survivra à travers le script et les mises à jour de JAWS. Dans ce document, la touche de préfixe s'appellera Commande; donc, par défaut, Commande 1; signifie qu'il faut frappez crochet ouvert [ suivi de 1.

Les scripts fournissent les commandes suivantes:

  • Commande a: déplace le focus sur la liste des appels actifs lorsque cela est possible. S'il n'y a pas d'appels actifs ou si la liste n'est pas affichée, cette commande ne fera rien.
  • Commande e: déplace le focus sur le champ de saisie de discussion (chat en anglais) pour la conversation affichée, s'il y en a une.
  • Commande h: déplace le focus sur la liste de l'historique des messages de conversation affichée.
  • Commande s: déplace le focus sur le bouton Afficher ou masquer la barre latérale dans un écran d'appel actif. Cela provoquera également l’annonce du nombre de notifications de conversation non lues, le cas échéant. Ouvrir la barre latérale, puis Tab ou Maj+Tab afin de parcourir l'écran pour accéder aux conversations et à d'autres informations pendant un appel.
  • Commande t: annonce qui est entraint d'écrire dans la conversation affichée.
  • Commande Maj+t: bascule entre activer ou désactiver l'inclusion d'informations sur l'heure du message pendant la lecture des messages de discussion.
  • Commande w: ouvre la page d'accueil de ces scripts (en anglais) dans le navigateur Web par défaut.

Les scripts fournissent également les fonctionnalités suivantes:

  • TOUCHE JAWS+Q, tout au long avec l'annonce du nom de la configuration active, annoncera le numéro de révision de ces scripts.
  • En tapant Insert+F1 deux fois rapidement, ouvrira ce document.
  • Plusieurs fonctionnalités du script peuvent être réglées via les éléments en haut de la boîte de dialogue classique Paramètres rapides de JAWS, ouvert avec TOUCHE JAWS+V. Lors de cette rédaction, ceux-ci comprennent
    • Que ce soit pour annoncer les messages entrants à l'arrivée dans la fenêtre en cours (bien que cette fonctionnalité ne soit pas encore implémentée).
    • Que ce soit pour annoncer et / ou jouer un son lorsque les participants à la conversation sont en train d'écrire.
    • Que ce soit pour annoncer les messages envoyer et modifier les informations de l'heure lors de la lecture des messages de discussion (chat en anglais).
    • Si et comment signaler les réactions à des messages de discussion spécifiques.
    • Que ce soit pour accélérer la navigation dans les listes en préservant la sélection au lieu de laisser Skype UWP le modifier sur chaque touche fléchée.
  • Les annonces de messages de discussion sont simplifiées de plusieurs manières et peuvent parfois inclure plus d'informations que celles normalement incluses par JAWS, telles que les informations de réaction lorsqu'elles sont présentes.
  • Lorsque le focus est dans le clavier numérique de l'appel entrant, ouvert via une option du menu "Autres fonctions d'appel", les scripts vont produire des chiffres. 0 jusqu'à 9, aussi bien que # et *, appuyer sur les boutons correspondants du clavier numérique pour provoquer la numérotation. Les scripts permettront également d'utiliser les lettres pour composer, en fonction des lettres affichées sur chaque bouton du clavier numérique; par exemple, en tapant b composera le numéro 2. Le support pour # et * peut être spécifique aux paramètres de langue et de clavier, Paramètres système et dépend de la disposition du clavier.
  • La commande JAWS SayAll fonctionne dans une liste de messages de discussion. C'est un moyen rapide de rattraper un groupe de discussion très occupé.
  • Par défaut, JAWS annoncera et jouera un son de frappe lorsqu'un ou plusieurs participants à une conversation sont en train d'écrire. Les annonces et les sons peuvent être basculé entre activé ou désactivé individuellement dans la boîte de dialogue Paramètres rapides de JAWS accessible via TOUCHE JAWS+V.
  • Commande t: annonce qui est entraint d'écrire.
  • JAWS gère plusieurs contrôles plus efficacement que ce ne serait le cas, y compris
    • Contrôles de liste (pas de répétition supplémentaire de l'élément étant laissé comme flèches de l'utilisateur).
    • Zones de liste déroulante (flèches annoncent plutôt que garder le silence).
    • Beaucoup de boutons bascule dans les paramètres qui, autrement, ne disent pas "non coché" quand ils sont désactivés.
    • Différents champs dont les noms sont nativement signalés de manière incorrecte.
  • La commande JAWS SayWindowTitle, Insert+T, lit les informations suivantes, dont seule la première partie serait lue nativement par JAWS:
    • Le nom de la fenêtre principale Skype UWP (généralement juste "Skype").
    • Informations sur l'appel actif s'il y en a un, y compris la durée de l'appel, le statut muet du microphone, et si l'appel est en attente.
    • Le nom de la fenêtre émergente active ou de la section, si celle-ci est reconnue; Par exemple, "Paramètres".
    • Le nombre d'appels actifs le cas échéant. Cela inclut les appels en attente. Cela peut également inclure des conversations de groupe qui peuvent devenir un appel même si elles ne le sont pas actuellement. ((Accédez aux autres appels en ouvrant la barre latérale via son bouton puis ensuite en recherchant et en parcourant avec les flèches directionnelles ou Ctrl+flèches directionnelles à travers de la liste d'appels actifs.)
    • Le texte de la zone de notification si un texte y est affiché. Exemple: "Mise à jour des conversations"
    • Une indication de la conversation affichée, si une conversation est affichée. Cette indication inclut le nom du contact, statut connecté lorsque disponible, et message d'humeur. Si la page d'accueil est affichée au lieu d'une conversation, JAWS indiquera "home" (accueil en français).
    • Si applicable à une conversation de groupe affichée, une indication si vous avez quitté le groupe.
  • En tapant Ctrl+TOUCHE JAWS+V trois fois rapidement fait apparaître une vue virtuelle JAWS répertoriant toutes les versions de Skype UWP vu sur cette machine pendant que JAWS et ces scripts fonctionnaient au fil du temps. ((JAWS lui-même fait en sorte qu’une simple pression sur cette commande annonce les informations de version et une double pression la virtualise.) Chaque version est affichée avec la date à laquelle elle a été vue pour la première fois par les scripts. Cela peut aider à diagnostiquer les problèmes et à déterminer s'ils représentent des bogues de script ou des problèmes causés par une mise à jour silencieuse de Skype UWP vers une version non encore prise en charge.

Conseils utiles

Quand un appel arrive dans Skype UWP, signalé par le son de l'appel entrant, appuyant sur Windows+V met le focus sur sa notification. JAWS annoncera les informations d'identification du correspondant (Autrement dit l'appelant). Depuis là, Tab au bouton que vous souhaitez utiliser en réponse (Audio pour un appel audio uniquement, Vidéo pour un appel audio / vidéo, Ignorer pour ignorer l'appel, etc.).).

Utiliser Alt+1 pour déplacer le focus sur la liste de conversation. Cette commande peut également être déplacée vers la première conversation de cette liste, mais sans basculer à cette conversation. C'était Alt+2 dans le Skype classic pour Windows Desktop.

Appuyez sur Entrée sur une conversation dans la liste de conversation pour afficher la conversation et placer le focus dans le champ de saisie correspondant.

Utiliser Alt+2 pour ouvrir la liste de contacts. C'était Alt+1 dans le client Skype classic pour Windows Desktop. Puisque la liste de contacts est recherchée par la boîte de dialogue "Rechercher dans Skype" (rapidement accessible avec Ctrl+F), cela peut rarement être nécessaire d'ouvrir la liste de contacts directement.

Un moyen rapide de commencer une nouvelle conversation:

  • Appuyez sur Ctrl+F pour y déplacer le focus sur la boîte de dialogue "Rechercher dans Skype".
  • Tapez le nom ou le nom partiel de votre contact prévu, et ensuite Tab deux fois pour atteindre les résultats de la recherche.
  • Utiliser les flèches Haut et Bas pour trouver votre contact prévu. Si, au début, vous ne trouvez pas le contact, vous parcourez à nouveau les résultats car la liste peut être mise à jour au fur et à mesure que d'autres correspondances sont trouvées.
  • Appuyez sur Entrée pour ouvrir une conversation et déplacer le focus dans le champ de saisie.
  • Pour appeler le contact immédiatement une fois la conversation affichée, tapez Ctrl+Maj+P pour un appel vocal ou Ctrl+Maj+K pour un appel vidéo.

Par défaut, ces scripts modifient le comportement des flèches et des autres touches de navigation dans certaines listes. Si cette fonctionnalité de script n'est pas désactivée, les flèches Origine, et Fin déplacera le focus mais sans changer la sélection et sans altérer les autres parties de l'affichage. La réponse native de Skype UWP à ces touches consiste à sélectionner chaque élément de la liste à son tour et mettre à jour l'affichage en conséquence. Cela peut être très lent, peut entraîner l'effacement par inadvertance des indicateurs de message non lus, et dans la liste des appels actifs peut envoyer le focus hors de la liste et compliquer ainsi la navigation dans la liste. Lors de cette rédaction, cette fonctionnalité s'applique par défaut aux listes suivantes:

  • La liste de conversation.
  • La liste d'appels active.
  • Les listes de résultats de la recherche.

Si vous choisissez de désactiver la fonctionnalité ci-dessus "préserver la sélection de liste" dans les Paramètres rapides, vous pouvez toujours vous déplacer dans les listes concernées sans modifier la sélection par l'utilisation de Ctrl+Bas et Ctrl+Haut plutôt que juste Bas et Haut par eux-mêmes. Cela déplacera le focus parmi les éléments de la liste sans modifier la sélection. Pour sélectionner l'élément focalisé, appuyer sur Espace.

Les utilisateurs avancés de JAWS peuvent également souhaiter examiner la section "Navigation rapide dans la liste de conversation avec le curseur tactile de JAWS (Avancé" pour un autre moyen efficace de naviguer dans la liste de conversation.

Avertissement: Les raccourcis clavier Ctrl+Maj+P et Ctrl+Maj+K pour faire des appels vocaux et vidéo appellera le contact affiché, pas le contact dont l'entrée est focalisée sur la liste de conversation s'il s'agit d'un contact différent. Pour être sûr que vous êtes en train d'appeler le contact correct tout en naviguant dans la liste de conversation, avec Ctrl+Haut et Ctrl+Bas, appuyez sur Espace avant de faire un appel.

Le curseur tactile de JAWS, activé en maintenant enfoncé la touche Maj tout en exécutant la commande classique de JAWS CurseurPC, il peut être utile dans certaines situations pour déterminer le contexte d'un champ. Dans les écrans de Paramètres, par exemple, essayez d’activer le curseur tactile, puis utilisez les touches fléchées Gauche et Droite pour explorer les éléments d’écran voisins. Les noms de champs apparaîtront parfois à gauche du champ actuel dans ce mode de navigation; et pour les champs qui ont un texte descriptif supplémentaire, il a tendance à apparaître à droite dans ce mode de navigation.

Lorsque vous utilisez le bouton Sélecteur de réaction sur un message de discussion, utilisez les flèches Gauche et Droite pour se déplacer parmi les choix de réaction, pas Haut et Bas. Dans certaines versions de Skype UWP, une pression sur la touche Tab est nécessaire après avoir appuyé sur Espace sur le bouton "Sélecteur de réaction" pour atteindre le premier bouton de réaction.

Navigation rapide dans la liste de conversation avec le curseur tactile de JAWS (Avancé

Cette sous-section décrit une technique très avancée de gestion de la liste de conversations Skype et n'est pas recommandée pour les nouveaux utilisateurs de JAWS. Il est également probablement inutile sauf sur les machines avec des historiques de conversation très volumineux.

Un exemple de quand la technique suivante est utile: Vous lancez Skype UWP et, à votre surprise, il y a 20 conversations avec des messages non lus. Ils ne sont toutefois pas tous regroupés en haut de la liste. (Cela peut se produire dans tous les clients Skype de Windows lorsque des appels sont arrivés depuis un téléphone utilisant un numéro SkypeIn.) Tenter de naviguer dans la liste avec les flèches est très lent; et même en utilisant Ctrl avec les flèches n'améliore pas significativement les performances.

La procédure suivante utilisera le curseur tactile de JAWS pour se déplacer parmi les éléments de la liste de conversation sans modifier la sélection ou mettre le focus à l'intérieur de la liste. D'abord, un peu de contextualisation:

JAWS est livré avec un "curseur tactile" parfois utile pour naviguer dans les applications Universal Windows Platform (UWP). Ce curseur peut opérer dans deux modes: En mode normal, les touches Gauche et Droite agissent comme des gestes de "balayage" ou "pichenette" sur les téléphones modernes, pour se déplacer à travers les éléments de l'écran dans une séquence logique qui tente d'inclure tous les éléments disponibles. En mode avancé, les éléments sont présentés dans une structure arborescente, où les touches Gauche et Droite se déplacent parmi les éléments au niveau de l'arborescence et Haut et Bas se déplacent parmi les niveaux. Cette sous-section se concentre sur l'utilisation de ce mode avancé pour contourner un problème de lenteur avec la liste de conversations Skype UWP.

Pour vous déplacer rapidement dans les entrées de la liste de conversation, suivez cette procédure:

  1. Se déplacer dans la liste de conversation en tapant Alt+1. Si nécessaire, appuyez sur la touche Origine pour se déplacer au premier élément de la liste.
  2. Activez le curseur tactile de JAWS en maintenant enfoncée la touche Maj tout en exécutant la commande normale CurseurPC. Le curseur tactile démarrera sur l'élément actuellement focalisé dans la liste de conversation.
  3. Assurez-vous que le mode avancé du curseur tactile est activé en appuyant sur la touche * sur le pavé numérique. Cette touche permet de basculer entre les modes avancé et normal.
  4. Utilisez les touches fléchées Gauche et Droite pour se déplacer à travers la liste des conversations. La réponse à chaque navigation doit être instantanée dans ce mode. Toute conversation avec des messages non lus sera indiquée lorsque vous naviguez.
  5. Pour mettre le focus et sélectionner une conversation, lire ou simplement effacer des messages non lus, appuyez sur Espace. Le focus restera sur l'élément que vous avez focalisé, ainsi vous pouvez continuer à naviguer vers d'autres conversations si vous préférez.
  6. Une fois cette procédure terminée, désactivez le curseur tactile tout en exécutant la commande CurseurPC deux fois rapidement.

Problèmes connus

Les problèmes présentés dans cette section peuvent être résolus dans les futures versions de script et / ou Skype UWP mais ne sont pas traités à ce jour.

L'opinion de cet auteur est la suivante: il existe de graves lacunes dans toutes les versions suivantes de Skype UWP: 11.19.0.856, 12.7.0.597, 12.8.0.480: 12.8.0.487:

  • La saisie des chiffres ne fait pas partie de la numérotation du clavier numérique; il faut cliquer ou appuyer sur Espace sur les touches numériques du clavier numérique. C'est fastidieux pour les utilisateurs de clavier uniquement pour des tâches telles que la saisie d'un code d'authentification par téléconférence à dix chiffres. Ces scripts rendent ce travail si possible pour le moment.
  • La redirection de la sortie audio de Skype UWP vers un périphérique audio autre que celui par défaut redirige l'appel audio, mais pas les sons de Skype tels que les sonneries.
  • Lorsque Skype UWP devient inactif après une période d'inactivité, son interface disparaît jusqu'à ce que la fenêtre soit à nouveau focalisée. La récupération à partir de cet état peut prendre plusieurs secondes ou une minute dans l'expérience de cet auteur, au cours de laquelle JAWS risque de ne plus répondre. La durée requise peut varier considérablement en fonction du matériel, des autres tâches en cours, de la taille de l'historique de Skype, etc. Si vous rencontrez ce problème, une solution de contournement possible est Alt+Tab afin de sortir de la fenêtre Skype UWP lorsque vous réalisez que tout n'est pas encore disponible, puis à revenir une minute plus tard après la restauration de l'interface.

Selon l'expérience de l'auteur du script, ces scripts peuvent être chargés depuis n'importe quelle autre application, parfois à la place des scripts pour cette application, lorsqu'une notification Skype arrive ou lorsque Skype termine une mise à jour de base de données volumineuse. La solution de contournement pour cela est Alt+Tab deux fois pour recharger les scripts corrects pour l'application focalisée. Ce problème semble s'appliquer aux apps UWP en général et constitue probablement un problème de JAWS.

Il n'y a pas le système habituel dans ces scripts pour naviguer parmi les messages de discussion (chat en anglais). Historiquement, cela a été fait avec Alt avec des chiffres pour lire les messages récents, et Alt avec les flèches, Origine, Find, et 5 du pavé numérique pour lire séquentiellement tout en gardant une trace de la position actuelle. La liste des messages de discussion (chat en anglais) de Skype UWP ne semble pas offrir ce type de navigation sans problèmes de performance et de fiabilité, selon de nombreuses expérimentations au cours du développement du script.

Certaines fenêtres émergentes et certains écrans intégrer n'ont pas de noms à l'écran et ne sont donc pas facilement identifiables par JAWS. Les exemples comprennent

  • Profils d'écrans.
  • La fenêtre émergente émoticônes (en appuyant sur "Choisir une émoticône").

JAWS devrait annoncer automatiquement les événements suivants au fur et à mesure qu'ils se produisent, mais ne le fait pas actuellement:

  • Messages de discussion entrants dans la fenêtre de discussion active.
  • L'apparence d'un message de notification dans l'app de Skype UWP, tel que "Mise à jour des conversations."
Les tentatives d'annonce des discussions entrantes avant la publication du script ont entraîné d'importants problèmes de performances et l'annonce de nombreux messages de discussion autres que ceux focalisés lors d'un défilement de l'historique des messages.

Dans Skype UWP 12.7.0.597 et 12.8.0.480 mais pas dans Skype UWP 11.19.0.856, présences en ligne (Connecté, Absent, Ne pas déranger) ne sont pas signalés aux lecteurs d'écran. Dans les versions 12.8.0.480 et .487, un autre indicateur indique parfois "actif maintenant" et toutefois des indications similaires.

JAWS peut dire "zone de liste" à plusieurs reprises lorsque le focus arrive sur certains contrôles de liste.
